Contexte du projet

Pour ce projet, nous avons été répartis en groupes. L’objectif était de réaliser une API REST (REpresental State Transfer) symfony permettant de gérer des Stages, des Organisations et des Contacts. Une API est une interface qui permet a des sytèmes logiciel de communiquer entre eux grâce au protocole HTTP, grâce a cette API dans notre contexte nous pouvons créer (POST) et récupérer (GET) un stage et un contact, nous pouvons aussi modifier et récupérer une organisation grâce a des routes.

Voici a quoi ressemble une API REST, il n'y a pas d'interface graphique et le résultat est présenter en JSON, le JSON est un format principalement utilisé pour stocker et transporter des données.

Pour plus de simplicité nous allons passer par une extension Google Chrome qui vas nous permettre de faire plus facilement des requêtes spécifiques et de les paramétrers plus facilement.
Talend cette extension permet de faire des requêtes HTTP. Nous pouvons ajouter ce que l'on appelle des assertions, elles permettent de faire des vérifications sur le résultat de la rêquete que nous avons effectué ainsi nous pouvons effectuer des testes pour vérifier que notre requête renvoie bien ce que nous voulons.

La requête effectuer ci-dessus est une requête GET, elle nous permet de récupérer tous les stages présents dans notre base de données - contruite avec notre ORM (Object Relation Mapper) permet de gérer manipuler et de récupérer des tables de données de la même façon qu'un objet quelconque, tout en gardant le langage PHP. Nous n'étions que 2 dans le groupe nous avons donc pas pu implémenter l'entrée API pour les contacts.

Une fois le corps de la requête écrit nous avons plus qu'à envoyer et nous obtiendrons le résultat en slide 3
