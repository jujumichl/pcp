Projet de cartographie des travaux de Fougères

CSS • HTML • JavaScript • PostGreSQL • PostGIS • Markdown

Application web: Goeserver, mapstore • Nombre de personnes sur le projet : 1 puis 2

Présentation du projet

Le projet a été réalisé par moi-même, puis Owen est m'a aider sur le projet au début de son stage. Le code du script JS est accessible sur GitHub : dépot GitHub

Origine et besoin client

Le projet provient d’une demande de la Mairie. La mairie souhaitait que les habitant puisse voir les travaux en cours et à venir dans la ville de fougères, ainsi qu'avoir l'impact des travaux.

Phase de conception

Le projet avais déjà une première base avec une cartographie des travaux de Fougères faite, sauf que le projet n'était pas assez documenter j'ai donc essayer de comprendre comment fonctionne le projet pour pouvoir y ajouter des modifications que Fabrice m'a demander. Cependant le manque de documentation m'a obligé à tout d'abord à reprendre le projet de zéros, ce que j'ai fait, je me suis donc renseigner sur les différents logiciels, et j'ai penser a trouver une façons de rendre stable la carte, j'ai donc penser a créer une base de données qui allais contenir les données nécessaire à la cartographie.

Développement – Script de synchronisation API base de données

Le script de synchronisation de la base de données (BDD) avec l'application programming Interface (API) était le plus gros du travail, nous avions a gérer une connexion. Voici ci-dessous le site web mis en place pour publier la cartographie des travaux de la mairie de Fougères.

Explication de la structure du projet

Le projet est structurer sur 5 applications, tout d'abord l'API qui vas nous permettre de récupérer les données, ensuite la base de données PostgreSQL/PostGIS qui vas contenir les données de l'API et ainsi permettre a notre application d'avoir une stabilité, puis nous avons geoserver qui vas interpréter les données qui sont stockée en base de donnée, et mapstore qui vas afficher les données interpréter par geoserver, enfin nous avons notre interface web. Pour garder a jour notre base de données nous avons du faire un script de lancement automatique du script de synchronisation puis automatiser le lancement du lancement de ce script.
