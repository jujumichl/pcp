Projet de site web de QCM

PHP Symfony • Stimulus • Webpack Encore • MySQL • Twig • Markdown

Framework UI : Bootstrap • Nombre de personnes sur le projet : 2

Présentation du projet

Le projet a été réalisé par Owen ainsi que moi-même, le code est accessible sur GitHub : dépot GitHub

Origine et besoin client

Le projet provient d’une demande de Yasin, employé du service informatique. Il souhaitait disposer d’un site permettant de créer et d’effectuer des QCM.

Phase de conception

Un brainstorming a été réalisé afin de définir les fonctionnalités et le design. Cette phase s’est déroulée sur une semaine et a permis une première mise en place de l’interface utilisateur (UI).

Vous pouvez cliquer sur l'image afin de l'ouvrir dans votre navigateur

Développement – Authentification

Une fois la maquette validée par Yasin, nous avons commencé le développement. L’authentification est liée à l’Active Directory (AD), ce qui ne nécessite pas la gestion du hashage des mots de passe. Cette partie a été mise en place par Owen.

Développement – Menu de gestion des QCM

Pendant qu’Owen s’occupait de la mise en place de la page de connexion, je me suis chargé du développement du menu latéral. Les fonctionnalités demandées par Yasin étaient :

Ci-dessous, plusieurs captures d’écran illustrent le fonctionnement du menu, vous pouvez cliquer dessus afin de les agendirs.

Voici la page d'accueil pour un nouvel utilisateur, il y a un texte qui permet a l'utilisateur de comprendre comment créer un nouveau QCM

Voici le menu de modification des QCM, ici l'utilisateur peux ajouter, renommer et supprimer un QCM

Voici l'interface quand un utilisateur choisit de renommer un QCM, comme vous pouvez le constater il y a un bouton de validation de renommage, il est aussi possible de valider le renommage avec la touche entrée et d'annuler le renommage avec la touche échap

Le message affiché est un message de confirmation de renommage, il apparait uniquement quand on clique sur un autre renommage ou qu'on souhaite quitter le mode de modification.

Il y a aussi la possibilité de séléctionner tous les QCM créer et donc de les supprimés.

Lorsque l'on souhaite supprimer un QCM, un message de confirmation apparait, ce message est la pour éviter les suppressions accidentelles

Une fois les messsages supprimés ils sont placer dans la corbeille, ils sont daté de la date de suppression, les QCM sont garder en corbeille pendant 7 jours, ils sont donc récupérable dans l'intervale des 7 jours suivant la suppression.

Un QCM séléctionner a un style différent que les QCM qui ne sont pas sélectionner afin de les différencier (ici la fonctionnalité qui vas servir a lire les QCM n'est pas encore développer).

Et voici la page d'accueil quand l'utilisateur a déjà créer des QCM

Les QCM peuvent être passer grâce a un code, ce code est un chiffre hexadécimal de 7 caractère de long, il est automatiquement générer a la création d'un QCM.
